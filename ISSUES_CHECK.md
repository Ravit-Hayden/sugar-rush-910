## 🚨 500 오류 관련 문제 (파일을 못 찾는 문제)

### 1. 캘린더 이벤트 추가 페이지
- **문제**: `/calendar/new` 페이지가 삭제됨
- **위치**: `src/routes/calendar/+page.svelte:58`
- **상태**: `handleAddEvent()` 함수에서 `goto('/calendar/new')`가 주석 처리됨
- **영향**: "새 일정 추가" 버튼 클릭 시 아무 동작도 하지 않음
- **필요 작업**: `/calendar/new` 페이지 생성 또는 모달/인라인 폼으로 대체

## 🎨 디자인 규칙 위반 체크

### 1. 그림자(Shadow) 사용 금지
- **규칙**: 그림자나 입체효과는 사용하지 않음
- **체크 필요**: `box-shadow`, `shadow-lg`, `shadow-md` 등 사용 여부 확인

### 2. 두꺼운 테두리 금지
- **규칙**: 두꺼운 테두리 사용 금지
- **체크 필요**: `border-2`, `border-4` 등 사용 여부 확인

### 3. 브랜드색 사용 규칙
- **규칙**: 브랜드색은 로고/악센트 전용, 앨범 대표색으로 사용 금지
- **체크 필요**: 앨범 관련 UI에서 `bg-brand-pink`, `text-brand-pink` 남용 확인

### 4. Pretendard 폰트
- **규칙**: Pretendard 웹폰트를 설치 후 기본으로 사용
- **체크 필요**: `src/app.css` 또는 전역 스타일에서 폰트 설정 확인

### 5. Lucide 아이콘 규칙
- **규칙**: Lucide 아이콘을 다운로드해서 사용, 16px, 컨테이너 24×24
- **체크 필요**: 
  - 아이콘 크기가 16px인지 확인
  - 컨테이너가 24×24인지 확인
  - `size={20}`, `size={24}` 등 규칙 위반 확인

### 6. 테마 토글 아이콘
- **규칙**: Lucide **Eclipse** 아이콘 1개 사용
- **체크 필요**: `ThemeToggle.svelte`에서 Eclipse 아이콘 사용 확인

### 7. 레이아웃 규칙
- **규칙**: 
  - Header 56px 고정
  - Sidebar 240px 토글
  - Main 12-col grid
  - 카드 h-72, KPI만 h-80
  - Right panel 없음
- **체크 필요**: 각 컴포넌트에서 규칙 준수 여부 확인

### 8. 반응형 규칙
- **규칙**: 
  - <md: 1열
  - md~<lg: 2열
  - ≥lg: 3열
  - 카드당 최대 4행, "더보기" 링크
- **체크 필요**: 그리드 레이아웃과 카드 행 제한 확인

### 9. 접근성 규칙
- **규칙**: 
  - WCAG 2.2 AA 대비 (≥4.5:1)
  - 포커스 링 ≥1.5px
  - aria-label/aria-live 적용
- **체크 필요**: 버튼, 링크, 입력 필드의 접근성 속성 확인

### 10. 검색어 하이라이트
- **규칙**: 검색어가 같을 때 색상도 매치
- **체크 필요**: 드롭다운 검색 결과에서 하이라이트 기능 확인
