# 보컬(Vocals) 페이지 분석

SUNO 보컬 페이지(`/suno/vocals`)의 현재 상태, 수정 사항, 보완·기능 제안을 정리한 문서입니다.

---

## 1. 적용된 수정 사항 (이번 반영)

| 항목 | 내용 |
|------|------|
| **아바타** | 그라데이션(`from-brand-pink to-purple-600`) 제거 → 미니멀: `bg-surface-1 border border-border-subtle text-text-strong` 원형 이니셜 |
| **보더** | 카드 하단 액션 영역 보더가 카드 양쪽 끝까지 이어지도록 `-mx-5 px-5 pt-3 border-t` 적용 |
| **디자인 규칙** | `BUTTON_STYLES_REFERENCE.md`에 “보더는 양쪽 패딩 없이 무조건 이어져야 한다” 규칙 추가 |

---

## 2. 수정해야 할 부분

| 우선순위 | 항목 | 현재 | 제안 |
|----------|------|------|------|
| **높음** | **수정 버튼 미동작** | 수정 버튼에 `onclick` 없음 | 수정 모달 열기 + 기존 이름/설명 로드 후 저장 시 `vocals` 업데이트 |
| **높음** | **연결 트랙 링크 없음** | “N개 트랙”만 표시, 클릭 불가 | 트랙 목록 페이지로 필터 링크(`?vocal=id`) 또는 트랙 ID 목록 상세 페이지로 이동 |
| **중간** | **추가 모달 버튼 스타일** | 제출 버튼에 `page-header-primary-button` 등 공통 클래스 미적용 | `BUTTON_STYLES_REFERENCE.md` 기준 Primary/Outline 통일 |
| **중간** | **취소 버튼** | `cancel-button`만 사용 | `btn-outline-hover` + `text-text-base` 등 보조 버튼 규칙 적용 |
| **낮음** | **정렬 드롭다운 선택 스타일** | 선택 시 `bg-brand-pink text-white` | 라이트 모드 대비 `text-text-base` 등과 대비되는지 확인 후, 필요 시 `focus:ring` 등 접근성 정리 |

---

## 3. 추가 보완 가능한 부분

| 영역 | 내용 |
|------|------|
| **카드 일관성** | 프롬프트/워드 카드와 동일하게 카드 하단 레이아웃(좌측 링크/우측 액션), `min-h-[240px]`, `flex-col mt-auto` 등 통일 검토 |
| **빈 상태** | “연결 트랙 0개”일 때 “트랙에 연결하기” 같은 안내 또는 링크 추가 |
| **검색/정렬** | 검색 초기화·정렬 옵션을 프롬프트 페이지와 동일한 패턴(예: `status-filter-btn`, `dropdown-item`)으로 통일 |
| **접근성** | 모달 `aria-labelledby`/`aria-describedby`, 정렬 드롭다운 `aria-activedescendant` 등 점검 |
| **데이터** | 현재 목업만 사용 → 실제 API/스토어 연동 시 `VirtualVocal[]` 로드·저장·삭제 흐름 정리 |

---

## 4. 기능 제안 (보컬 음원 포함)

| 기능 | 설명 |
|------|------|
| **보컬 음원(데모) 첨부** | “보컬은 음원도 추가될 수 있어” 요구 반영. `VirtualVocal`에 선택 필드 추가 예: `demoAudioUrl?: string` 또는 `demoAudioFileId?: string`. 카드에 미리듣기 버튼/플레이어, 추가·수정 모달에 음원 업로드 또는 URL 입력 |
| **수정 모달** | 추가 모달과 유사한 레이아웃으로 “보컬 수정” 모달 구현. 이름·설명·(선택) 음원 수정 후 저장 |
| **연결 트랙 보기** | “N개 트랙” 클릭 시 해당 보컬이 연결된 트랙 목록 페이지로 이동(쿼리 `vocal=id`) 또는 슬라이드 패널로 트랙 목록 표시 |
| **삭제 확인** | 연결 트랙이 있을 때 확인 메시지 유지, “연결 해제 후 삭제” vs “그대로 삭제(연결만 유지)” 옵션 검토 |
| **정렬** | “최근 사용순” 등 사용 빈도 기반 정렬은 데이터 연동 후 추가 가능 |

---

## 5. 타입 확장 (적용됨 / 제안)

**적용됨 – 보컬 사진**

- `VirtualVocal`에 `imageUrl?: string` 추가.
- 추가 모달에 "사진 URL (선택)" 입력란 추가. URL 있으면 카드에 `<img>`로 표시, 없으면 이름 첫 글자만 표시.

**제안 – 보컬 음원**

```ts
// VirtualVocal 예시 (음원 추가 시)
demoAudioUrl?: string;   // 또는 demoAudioFileId
```

- 음원 필드는 선택(`?`)으로 두고, 기존 목업/API와 호환 가능하게 두는 것을 권장합니다.

---

## 6. 요약

- **즉시 반영된 것**: 아바타 미니멀화, 보더 풀폭, 디자인 규칙 문서화.
- **우선 수정 권장**: 수정 버튼 동작, 연결 트랙 링크, 모달·버튼 스타일 통일.
- **보완 권장**: 카드/빈 상태/검색·정렬·접근성 일관성, 데이터 연동.
- **기능 확장**: 보컬 음원(데모) 필드·UI, 수정 모달, 연결 트랙 보기.

이 문서는 보컬 페이지를 단계적으로 개선할 때 체크리스트로 사용할 수 있습니다.
