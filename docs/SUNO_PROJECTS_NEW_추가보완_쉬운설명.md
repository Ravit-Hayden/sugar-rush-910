# SUNO 새 프로젝트 페이지 – 추가 보완 (쉬운 설명)

아래는 “나중에 여유 있을 때 하면 좋은 것들”을 **쉽게** 정리한 내용입니다.

---

## 1. 포커스 링이 부드럽게 나타나게

- **지금:** 아코디언 제목을 Tab으로 포커스하면 핑크 링이 바로 생김.
- **보완:** 링이 0.15초 정도 서서히 나타나게 하면 덜 튀어 보임.
- **필수 아님:** 그냥 두어도 동작에는 문제 없음.

---

## 2. Simple 모드일 때 안내 문구

- **지금:** “프롬프트 · 가사” 섹션에서 **Simple** 모드면 가사 입력 칸이 안 보임.
- **보완:** Simple일 때 “스타일만 입력하는 모드예요” 같은 짧은 설명을 한 줄 넣어 주기.
- **효과:** 사용자가 “가사 칸이 왜 없지?” 하고 헷갈리지 않음.

---

## 3. 모바일에서 미리보기/제작 단계

- **지금:** 화면이 좁으면 오른쪽 “미리보기·제작 단계”가 아래로 내려감. 위에 고정(sticky)은 큰 화면에서만 의미 있음.
- **보완:** 모바일에서는 sticky를 끄거나, “미리보기 접기” 버튼을 두는 식으로 정리할 수 있음.
- **필수 아님:** 지금처럼 둬도 사용은 가능함.

---

## 4. 에러 메시지 위치

- **지금:** 잘못된 입력이 있으면 상단 빨간 박스에 한 줄로 표시.
- **보완:** 나중에 “제목 필수”, “Styles 1000자 초과”처럼 **필드별**로 해당 입력칸 근처에 빨간 메시지를 띄우면 더 찾기 쉬움.
- **시점:** 필드가 많이 늘어나면 그때 맞춰 적용해도 됨.

---

## 5. 제출 버튼 주변 (모바일)

- **지금:** “취소” / “프로젝트 생성”이 가로로 나란히 있음.
- **보완:** 화면이 매우 좁을 때는 세로로 쌓이거나, 버튼을 더 크게 보이게 하는 식으로 조정 가능.
- **필수 아님:** 지금도 눌리긴 함.

---

## 6. 기타 (정책/운영 쪽)

- **커버 이미지:** GIF 허용할지, 서비스 정책에 맞게 정하면 됨.
- **제작 단계:** “첫 3단계 자동 포함”을 실제 저장 API에 어떻게 넣을지 나중에 정하면 됨.
- **템플릿 이름:** URL에 잘못된 템플릿 이름이 오면 “없는 템플릿이에요” 같은 안내를 넣을지 결정하면 됨.

---

## 요약

- **당장 꼭 할 것:** 없음. 이미 적용한 보완만으로도 사용·접근성·검증은 충분함.
- **나중에 하면 좋은 것:**  
  1) 포커스 링 전환,  
  2) Simple 모드 안내 문구,  
  3) 모바일 미리보기/제작 단계,  
  4) 필드별 에러 표시.
- **정책/API 정리 후:** 커버 형식, 제작 단계 연동, 템플릿 에러 메시지.
